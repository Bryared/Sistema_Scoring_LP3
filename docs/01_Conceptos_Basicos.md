# Capítulo 2: Glosario y Conceptos Financieros

Este capítulo traduce la terminología bancaria y técnica utilizada en el sistema para que cualquier usuario o auditor pueda entender la lógica detrás del código.

## 2.1. Conceptos Bancarios Core
- **Onboarding:** El proceso digital por el cual una persona anónima se convierte en cliente oficial del banco. Es el momento de mayor riesgo de fraude.
- **Scoring Crediticio:** La "calificación" o nota de confianza de un usuario. Define si es apto para recibir dinero prestado.
- **DSR (Debt Service Ratio / Carga Financiera):** La métrica más importante de insolvencia. Mide qué porcentaje de los ingresos mensuales de un cliente ya están comprometidos en pagar deudas existentes. Si el DSR supera el 35%, el cliente está en riesgo de impago (default).
- **Ruleteo:** Práctica financiera desesperada donde un cliente saca dinero de una Tarjeta de Crédito A para pagar la Tarjeta de Crédito B, creando una burbuja de deuda.
- **Smurfing (Pitufeo):** Táctica de Lavado de Activos. El criminal divide grandes sumas de dinero ilícito en pequeños montos y utiliza testaferros (mulas) para hacer transferencias rápidas en círculos, buscando ocultar el origen de los fondos.

## 2.2. Conceptos Tecnológicos
- **XAI (Inteligencia Artificial Explicable):** Algoritmos diseñados para que el humano entienda exactamente su proceso de toma de decisiones.
- **Prolog (Programación Lógica):** Un lenguaje de Inteligencia Artificial Simbólica basado en predicados matemáticos verdaderos o falsos. Excelente para compliance legal.
- **Hash Criptográfico (SHA-256):** Un algoritmo matemático que convierte cualquier texto en una cadena de caracteres única. Si una letra del texto original cambia, el Hash cambia por completo. Se usa para garantizar que los registros (Logs) no hayan sido alterados por humanos.
